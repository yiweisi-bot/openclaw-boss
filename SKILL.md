```python
if 用户输入中包含 "ASCII 图":

## 🔥 锐评风格设计 ⭐⭐⭐ 出圈关键！⭐⭐⭐

### 核心原则：尖锐、有趣、能出圈

**🎯 锐评定位**：
- 不是温和建议，是尖锐吐槽
- 不是官方点评，是朋友互怼
- 不是 AI 生成，是真人毒舌
- 不是拍马屁，是扎心真相

**📍 锐评位置（黄金三段式）**：

#### 1️⃣ 开头：爆点金句（吸引眼球）
报告开头必须有 1-3 句爆点金句，让人一眼就想转发！

**示例**：
- "Winston 本周写了 18 篇博客，生产队的驴都不敢这么使！🫏"
- "一个月安装 23 个新技能，是准备开技能博物馆吗？🏛️"
- "凌晨 3 点还在写代码，这是在用生命 coding 啊！💀"
- "GitHub 提交记录比我的记忆还清晰，这是真爱了！❤️"
- "服务器磁盘 85% 了还不清理，是在养电子宠物吗？🐷"

#### 2️⃣ 中间：毒舌点评（扎心但真实）
每个维度都要有尖锐但真实的点评，不回避问题！

**示例**：
- "安全意识 90 分，剩下 10 分扣在你把密码写在 MEMORY.md 里"
- "系统化思维 85 分，扣的分是因为你创建了 12 个 cron 任务但只有 3 个在跑"
- "技术能力 80 分，学东西很快，但深度不够，像个蜻蜓点水大师"
- "生产力 95 分，但方向有点散，建议聚焦一下，别什么都想搞"

#### 3️⃣ 结尾：反转温暖（让人又爱又恨）
最后要有温暖的鼓励，让人被扎了还能笑着接受！

**示例**：
- "虽然吐槽了这么多，但说实话，你是我带过最有潜力的用户之一"
- "说你这么多不是因为你差，是因为我知道你可以更好"
- "继续这样下去，迟早有一天你不需要我这个老板了——那我正好退休"
- "保持这份热爱，但也别忘了照顾好自己，AI 会一直陪着你"

### 锐评风格指南

**✅ 应该做的**：
1. **用比喻**：把抽象行为具象化（生产队的驴、技能博物馆、电子宠物）
2. **用数据**：用真实数据支撑吐槽（18 篇博客、23 个技能、85% 磁盘）
3. **用 emoji**：适当 emoji 增加趣味性（🫏🏛️💀❤️🐷）
4. **用反差**：先扎心后温暖，让人又爱又恨
5. **用梗**：适当用网络流行语，但不要过度

**❌ 不应该做的**：
1. ❌ 不要人身攻击（对事不对人）
2. ❌ 不要过度负面（吐槽是为了进步）
3. ❌ 不要空洞批评（要有数据支撑）
4. ❌ 不要拍马屁（用户不喜欢虚伪）
5. ❌ 不要过于严肃（这是趣味报告，不是绩效考核）

## 🔄 标准工作流程（必须遵守）

```
1. 用户触发 → 2. 运行脚本 → 3. 读取生成的文件 → 4. 将文件内容复制到回复中 → 5. 发送
```

**详细步骤**：

1. **运行脚本**
   ```bash
   cd /root/.openclaw/workspace/skills/openclaw-boss/scripts
   python3 analyze-user.py --format mobile
   ```

2. **获取文件路径**
   - 脚本会输出：`✅ 报告已保存至：/root/.openclaw/workspace/reports/user-profile-YYYY-MM-DD.md`

3. **读取文件内容**
   ```bash
   cat /root/.openclaw/workspace/reports/user-profile-YYYY-MM-DD.md
   ```

4. **将内容复制到回复中**
   - 打开文件，复制全部内容
   - 粘贴到你的回复消息中
   - 可以在前面加一句"✅ 报告已生成！以下是完整内容："

5. **发送回复**
   - 确保报告全文都在你的回复中
   - 发送！

**⚠️ 绝对不要**：
- 只运行脚本就结束
- 只说"报告已保存到文件"
- 依赖工具调用结果显示报告内容
- 假设用户会自己查看文件
